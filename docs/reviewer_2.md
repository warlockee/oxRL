Formatting vs. Mathematical ReasoningThe paper frames itself around "mathematical reasoning." Yet, Table 3 shows that on the MATH dataset (which requires much deeper reasoning than GSM8K), the performance spread between all methods collapses to a negligible 0.54 pp (a 36x compression). This heavily implies that the massive 19.3 pp variance on GSM8K isn't a difference in latent reasoning capabilities—it is almost entirely a difference in format compliance (getting the model to reliably output the \n#### [Answer] string required by GSM8K's strict-match parser). If SGRPO learns syntax better at 1.5B, while SimPO learns syntax better at 7B, this isn't a deep algorithmic scaling law; it's an artifact of GSM8K formatting sensitivity.6. The PyTorch "Bug" is Actually User ErrorFraming the DistributedSampler determinism as a "hidden PyTorch bug" is sensationalist. PyTorch documentation explicitly states that users must pass a seed/generator and call sampler.set_epoch(epoch) to ensure proper shuffling across workers in distributed training. Failing to do so is a framework implementation omission on the researcher's part, not a flaw in the PyTorch codebase itself.
