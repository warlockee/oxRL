# Reproducing the Paper's ~350 Training Runs

This document maps the experiments in [`docs/oxrl_formal.tex`](../docs/oxrl_formal.tex) to concrete commands, hyperparameters, and seeds. The intent is that an external reviewer can clone the repo, follow the matrix here, and arrive at the same per-seed numbers reported in Tables 1, 2, 6, 7, 18, 19 of the paper.

Per-seed result JSONs (one per `(model, algorithm, seed)` row) will be released at acceptance under `experiments/results/`. Until then this README + the config-generation script (`experiments/build_matrix.py` below) is the source of truth for what was run.

## 2. Hyperparameters (shared)

```yaml
optimizer: adamw
optimizer_params:
  betas: [0.9, 0.95]
  eps: 1.0e-8
weight_decay: 0.01
lr_schedule: cosine
warmup_ratio: 0.10
effective_batch_size: 8       # 1 prompt × 8 grad-accum
total_number_of_epochs: 3
precision: bf16
distributed_strategy: deepspeed_zero3
hardware: 8 × NVIDIA H100 80GB HBM3
sequence_length:
  math: 512
  code: 1024
```

Algorithm-specific:

| Algorithm | LR | Key hyperparameters | GPUs/run |
| --- | --- | --- | --- |
| SP-RFT | 1e-6 | — | 1 |
| DPO | 1e-6 | β=0.1 | 1 |
| SimPO | 1e-6 | β=0.1, γ=0.5 | 1 |
| IPO | 1e-6 | β=0.1 | 1 |
| KTO | 1e-6 | β=0.1 | 1 |
| SGRPO | 1e-6 | λ_KL=0, n=4 (group size) | 2 |
| GSPO | 1e-6 | λ_KL=0, n=4 | 2 |
| CISPO | 1e-6 | λ_KL=0, n=4, clip=[0.8, 1.2] | 2 |

LoRA at ≥7B: r=16, α=32, applied to all linear layers. Full FT below 7B and in the 7B factorial cell.

## 3. Per-seed running

The actual configs are generated by [`oxrl/swarm/config_generator.py`](../oxrl/swarm/config_generator.py). To regenerate the full matrix as a directory of YAML files, run [`experiments/build_matrix.py`](build_matrix.py):

```bash
python experiments/build_matrix.py --out experiments/configs
```

This emits one YAML per `(model, algorithm, seed)` triple, each callable directly:

```bash
oxrl train --config experiments/configs/qwen2.5-7b-instruct__dpo__seed42.yaml
```

For the DPO-variant sweep at 1.5B (100 runs):

```bash
for variant in dpo ipo simpo kto hinge cpo orpo rdpo cdpo betadpo \
               caldpo dpop odpo exo alphapo apo sppo robust_dpo gpo focalpo; do
  for seed in 42 123 456 789 1024; do
    oxrl train --config experiments/configs/qwen2.5-1.5b-instruct__${variant}__seed${seed}.yaml
  done
done
```

## 4. Evaluation

All evaluations go through [`oxrl/eval/evaluator.py`](../oxrl/eval/evaluator.py), which now wraps the LM Evaluation Harness for: `gsm8k`, `gsm8k_cot`, `math` (minerva_math), `mbpp`, `humaneval`, `arc_challenge`, `hellaswag`, `winogrande`. Reproduce the GSM8K + MATH + general numbers in the paper:

```bash
# GSM8K (8-shot, strict-match)
oxrl eval --checkpoint <ckpt> --tasks gsm8k

# MATH (4-shot)
oxrl eval --checkpoint <ckpt> --tasks math

# General-domain trio (Open LLM Leaderboard convention: 25/10/5 shot)
oxrl eval --checkpoint <ckpt> --tasks arc_challenge,hellaswag,winogrande
```

## 5. Compute budget

~940 GPU-hours total on H100 80GB (training: ~720 h; evaluation: ~220 h). Per-bucket breakdown is in Appendix I (Table 17) of the paper.

Per-checkpoint release plan (post-acceptance):
- `huggingface.co/<anonymized>/oxrl-qwen2.5-{0.5b,1.5b,3b,7b,14b}-{algo}-seed{seed}` for the core matrix.
- `huggingface.co/<anonymized>/oxrl-gemma3-{1b,4b,12b}-{algo}-seed{seed}` for the cross-architecture set.
- Per-seed eval JSONs at `experiments/results/<model>/<algo>/seed<seed>.json`.

These will be deanonymized and pushed at camera-ready.
